## 1. 系统简介

### 1.1 产品概述

源码安全审计系统是一个专业的源代码访问控制与审计解决方案，旨在帮助组织：

- **统一认证管理**：所有用户必须通过身份验证才能访问源码
- **操作审计追踪**：完整记录用户对源码的所有操作（读取、编辑、删除、下载）
- **细粒度权限控制**：基于角色和目录的权限管理
- **实时文件监控**：监控源码文件的变更并记录审计日志
- **可视化管理界面**：友好的 Web 管理界面，支持审计日志查询和分析

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 用户认证 | 支持用户名密码登录、Token 刷新、会话管理 |
| 用户管理 | 创建、编辑、删除用户，分配角色和权限 |
| 文件浏览器 | 浏览、查看、编辑、下载源码文件 |
| 审计日志 | 查看、筛选、导出操作审计记录 |
| 监控服务 | 实时监控文件变更并自动记录 |
| 系统设置 | 配置监控路径、安全策略等 |

## 3. 安装部署

### 3.1 后端安装

#### 步骤 1: 克隆或获取项目代码

```bash
cd /path/to/project
```

#### 步骤 2: 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

如果使用虚拟环境（推荐）：

```bash
# Linux/macOS
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Windows
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

#### 步骤 3: 配置系统

主要配置文件位于 [config/config.yaml](file:///workspace/config/config.yaml)：

```yaml
app:
  name: "Source Code Audit System"
  port: 8000
  debug: false

database:
  path: "./data/audit.db"

security:
  secret_key: "请修改为生产环境安全密钥"

monitor:
  enabled: true
  watch_paths:
    - "/path/to/your/source/code"
```

**重要配置项说明**:
- `app.port`: 后端服务端口
- `database.path`: SQLite 数据库文件路径
- `security.secret_key`: 生产环境必须修改为强随机密钥
- `monitor.watch_paths`: 要监控的源码目录列表（支持多个路径）
- `monitor.enabled`: 是否启用文件监控服务

#### 步骤 4: 启动后端服务

**方式 1: 使用启动脚本**

```bash
# Linux/macOS
cd /workspace
./start-backend.sh

# Windows
start-backend.bat
# 或
start-backend.ps1
```

**方式 2: 手动启动**

```bash
cd /workspace
export CONFIG_PATH="./config/config.yaml"  # Linux/macOS
# 或
set CONFIG_PATH=.\config\config.yaml      # Windows

python -m backend.app.main
```

或者使用 Uvicorn：

```bash
cd /workspace
uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
```

启动成功后，访问 `http://localhost:8000/health` 应该能看到健康检查响应。

### 3.2 前端安装

#### 步骤 1: 安装 Node.js 依赖

```bash
cd frontend
npm install
```

#### 步骤 2: 启动前端开发服务

```bash
npm run dev
```

前端将在 `http://localhost:3000` 启动。

#### 步骤 3: 构建生产版本（可选）

```bash
npm run build
```

构建产物将输出到 `frontend/dist` 目录。

### 3.3 验证安装

1. 访问 `http://localhost:3000`，应该能看到登录页面
2. 使用默认管理员账号登录
   - 用户名: `admin`
   - 密码: `Admin@123456`
3. 登录成功后，系统会要求你修改初始密码

## 4. 快速开始

### 4.1 首次登录

1. 打开浏览器访问 `http://localhost:3000`
2. 输入默认管理员凭证：
   - **用户名**: `admin`
   - **密码**: `Admin@123456`
3. 点击"登录"按钮
4. 首次登录后，系统会强制要求修改密码
   - 输入旧密码
   - 设置新密码（建议：至少 8 位，包含大小写字母和数字）
   - 确认新密码
   - 点击"修改密码"

### 4.2 创建第一个普通用户

1. 登录后，点击左侧导航栏的"用户管理"
2. 点击"新建用户"按钮
3. 填写用户信息：
   - 用户名（必填，唯一）
   - 邮箱（可选，唯一）
   - 初始密码
   - 角色（`user` 或 `admin`）
4. 点击"创建"

### 4.3 配置监控路径

1. 点击左侧导航栏的"系统设置"
2. 在"文件监控"部分，添加要监控的源码目录
3. 点击"保存设置"
4. 系统将自动开始监控这些目录的文件变更

### 4.4 浏览文件

1. 点击左侧导航栏的"文件浏览器"
2. 在左侧目录树中选择目录
3. 在右侧文件列表中查看和操作文件
4. 点击文件名可查看文件内容

## 5. 功能使用说明

### 5.1 用户认证

#### 登录

1. 访问登录页面
2. 输入用户名和密码
3. 点击"登录"
4. 登录成功后会自动跳转到仪表板

#### 修改密码

1. 登录后，点击右上角用户头像
2. 选择"修改密码"
3. 输入旧密码和新密码
4. 点击"确认"

#### 登出

1. 点击右上角用户头像
2. 选择"退出登录"

### 5.2 仪表板

仪表板提供系统概览：
- **今日操作统计**: 显示今日各类操作数量
- **最近活动**: 显示最新的审计日志
- **系统状态**: 显示监控服务状态和系统健康状况
- **快捷入口**: 快速访问常用功能

### 5.3 文件浏览器

#### 浏览目录

1. 进入文件浏览器页面
2. 左侧是目录树，点击展开/折叠目录
3. 右侧显示当前目录的文件列表
4. 支持按名称、大小、修改时间排序

#### 查看文件内容

1. 在文件列表中点击文件名
2. 文件内容将在预览区显示
3. 支持文本文件预览，大文件默认显示前 1000 行

#### 编辑文件

1. 找到要编辑的文件
2. 点击"编辑"按钮
3. 在编辑器中修改内容
4. 点击"保存"
5. 修改操作会自动记录到审计日志

#### 下载文件

1. 找到要下载的文件
2. 点击"下载"按钮
3. 文件将保存到本地
4. 下载操作会记录到审计日志

#### 删除文件

1. 找到要删除的文件
2. 点击"删除"按钮
3. 确认删除操作
4. 删除操作会记录到审计日志

**注意**: 删除操作不可恢复，请谨慎操作！

### 5.4 审计日志

#### 查看审计日志

1. 进入审计日志页面
2. 可以看到所有用户的操作记录
3. 每条记录包含：
   - 时间戳
   - 操作用户
   - 操作类型
   - 操作对象
   - 操作状态
   - 客户端 IP

#### 筛选日志

使用筛选条件查找特定日志：
- **时间范围**: 选择开始和结束日期
- **用户**: 筛选特定用户的操作
- **操作类型**: 筛选特定操作类型（登录、读取、编辑、删除等）
- **文件路径**: 按文件路径搜索
- **操作状态**: 成功/失败

#### 导出日志

1. 点击"导出"按钮
2. 选择导出格式（CSV、JSON）
3. 选择时间范围
4. 点击"导出"
5. 日志文件将自动下载

### 5.5 用户管理（仅管理员）

#### 查看用户列表

1. 进入用户管理页面
2. 显示所有用户列表
3. 可以看到用户状态、角色、创建时间等信息

#### 创建新用户

1. 点击"新建用户"
2. 填写用户信息：
   - 用户名（必填）
   - 邮箱（可选）
   - 初始密码
   - 角色（user/admin）
3. 点击"创建"

#### 编辑用户

1. 在用户列表中找到要编辑的用户
2. 点击"编辑"按钮
3. 修改用户信息
4. 点击"保存"

#### 停用/启用用户

1. 在用户列表中找到用户
2. 点击"停用"或"启用"按钮
3. 被停用的用户无法登录

#### 重置用户密码

1. 在用户列表中找到用户
2. 点击"重置密码"
3. 输入新密码
4. 点击"确认"

#### 删除用户

1. 在用户列表中找到用户
2. 点击"删除"按钮
3. 确认删除操作

**注意**: 删除用户不会删除该用户的审计日志记录。

### 5.6 系统设置（仅管理员）

#### 配置监控路径

1. 进入系统设置页面
2. 找到"文件监控"部分
3. 添加或删除监控路径
4. 点击"保存设置"

**建议**: 只监控必要的源码目录，避免监控临时文件、构建产物等。

#### 配置安全策略

1. 在"安全设置"部分配置：
   - Token 有效期
   - 登录失败锁定策略
   - 密码强度要求
2. 点击"保存设置"

#### 查看系统状态

在设置页面底部可以查看：
- 系统版本
- 数据库状态
- 监控服务状态
- 磁盘使用情况

## 6. 管理员操作指南

### 6.1 安全最佳实践

#### 1. 修改默认密钥

首次部署后，立即修改配置文件中的密钥：

```yaml
security:
  secret_key: "替换为强随机密钥"
```

生成强密钥的方法：

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

#### 2. 定期备份数据库

建议定期备份数据库文件：

```bash
# 备份
cp data/audit.db data/audit.db.backup.$(date +%Y%m%d)

# 恢复
cp data/audit.db.backup.20260514 data/audit.db
```

#### 3. 限制管理员数量

- 只给必要的人员分配管理员权限
- 普通用户使用 `user` 角色
- 定期审查用户权限

#### 4. 监控审计日志

- 定期查看审计日志
- 关注异常操作（如大量删除、非工作时间访问）
- 及时响应安全事件

### 6.2 故障排查

#### 后端服务无法启动

1. 检查端口是否被占用：
   ```bash
   lsof -i :8000  # Linux/macOS
   netstat -ano | findstr :8000  # Windows
   ```

2. 检查配置文件路径是否正确

3. 查看日志文件排查错误

#### 前端无法连接后端

1. 确认后端服务正在运行
2. 检查 [vite.config.js](file:///workspace/frontend/vite.config.js) 中的代理配置
3. 检查浏览器控制台的网络请求错误

#### 文件监控不工作

1. 确认 `monitor.enabled` 设置为 `true`
2. 检查监控路径是否存在且有读取权限
3. 查看后端日志确认监控服务状态

#### 数据库问题

1. 检查数据库文件权限
2. 如数据库损坏，从备份恢复
3. 可以删除数据库文件让系统重新初始化（会丢失所有数据）

## 7. 常见问题

### Q1: 忘记管理员密码怎么办？

**A**: 需要直接操作数据库重置密码。

```python
# 在 Python 中运行
from backend.app.database import SessionLocal
from backend.app.models.user import User
from backend.app.services.auth_service import get_password_hash

db = SessionLocal()
admin = db.query(User).filter(User.username == 'admin').first()
if admin:
    admin.password_hash = get_password_hash('NewPassword123')
    admin.must_change_password = True
    db.commit()
db.close()
```

### Q2: 如何修改默认端口？

**A**: 修改配置文件：

```yaml
app:
  port: 9000  # 修改为你想要的端口
```

同时修改前端 [vite.config.js](file:///workspace/frontend/vite.config.js) 中的代理目标端口。

### Q3: 支持 PostgreSQL 吗？

**A**: 是的，修改数据库配置：

```yaml
database:
  type: "postgresql"
  url: "postgresql://user:password@localhost/dbname"
```

### Q4: 审计日志保留多长时间？

**A**: 默认永久保留。可以定期归档旧日志或使用日志清理脚本。

### Q5: 如何迁移到生产环境？

**A**: 生产环境部署建议：

1. 使用 PostgreSQL 代替 SQLite
2. 配置 HTTPS
3. 使用 Nginx 反向代理
4. 配置防火墙规则
5. 设置定期备份
6. 启用日志轮转
7. 监控系统资源使用

### Q6: 可以同时监控多个目录吗？

**A**: 可以，在配置中添加多个路径：

```yaml
monitor:
  watch_paths:
    - "/path/to/project1"
    - "/path/to/project2"
    - "D:\\Projects\\SourceCode"  # Windows 路径
```
