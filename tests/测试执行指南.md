# 测试执行指南

## 一、环境准备

### 1.1 安装后端测试依赖

```bash
# 进入工作目录
cd /workspace

# 安装测试依赖
pip install -r tests/requirements-test.txt
```

### 1.2 创建测试数据库目录

```bash
mkdir -p test_data
```

## 二、运行测试

### 2.1 方式一：使用脚本（推荐）

#### Linux/macOS

```bash
# 进入测试目录
cd /workspace/tests

# 添加执行权限
chmod +x run_tests.sh

# 交互式菜单
./run_tests.sh

# 或使用命令行选项
./run_tests.sh --quick          # 快速测试
./run_tests.sh --with-coverage  # 生成覆盖率报告
```

#### Windows

```cmd
cd tests
run_tests.bat
```

### 2.2 方式二：直接使用 pytest

```bash
# 运行所有测试
pytest tests/ -v

# 运行特定模块
pytest tests/backend/test_auth.py -v
pytest tests/backend/test_users.py -v
pytest tests/backend/test_files.py -v
pytest tests/backend/test_audit.py -v

# 运行测试并显示详细输出
pytest tests/ -v -s

# 遇到第一个失败就停止
pytest tests/ -v -x

# 只运行失败的测试
pytest tests/ --lf
```

## 三、查看测试覆盖率

### 3.1 生成覆盖率报告

```bash
# 生成 HTML 覆盖率报告
pytest tests/backend/ --cov=backend --cov-report=html

# 查看报告
open htmlcov/index.html  # macOS
xdg-open htmlcov/index.html  # Linux
start htmlcov/index.html  # Windows
```

### 3.2 覆盖率配置

在 `pytest.ini` 中配置：

```ini
[coverage:run]
source = backend
omit =
    */tests/*
    */migrations/*
    */__pycache__/*
```

## 五、持续集成配置

### 5.1 GitHub Actions

创建 `.github/workflows/test.yml`：

```yaml
name: Test

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  backend-test:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.9'

    - name: Install dependencies
      run: |
        pip install -r requirements.txt
        pip install -r tests/requirements-test.txt

    - name: Run tests
      run: |
        pytest tests/backend/ -v --cov=backend --cov-report=xml

    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        files: ./coverage.xml
```

### 5.2 本地 CI 检查

在提交代码前运行：

```bash
# 运行所有测试
./tests/run_tests.sh --with-coverage

# 或快速检查
./tests/run_tests.sh --quick
```

## 六、常见问题

### Q1: 测试数据库被锁定？

```bash
# 删除测试数据库重新开始
rm -f test_data/test_audit.db
pytest tests/ -v
```

### Q2: 测试全部失败？

```bash
# 检查依赖是否完整
pip install -r requirements.txt
pip install -r tests/requirements-test.txt

# 验证配置文件
export CONFIG_PATH="./config/config.yaml"
pytest tests/ -v
```

### Q3: 覆盖率报告为空？

```bash
# 确保安装 coverage 插件
pip install pytest-cov

# 重新运行
pytest tests/ --cov=backend --cov-report=term-missing
```

## 七、测试最佳实践

### 7.1 编写新测试

1. 测试应该独立，每个测试函数创建自己的数据
2. 使用描述性的测试名称
3. 每个测试只验证一个功能点
4. 测试应该快速执行
5. 使用 mock 避免依赖外部服务

### 7.2 测试命名规范

```python
def test_<功能>_<场景>_<预期结果>(self, ...):
    """
    测试描述

    Given: 测试的前置条件
    When: 执行的操作
    Then: 预期的结果
    """
    pass
```

示例：

```python
def test_login_success_with_valid_credentials(self, client, test_user):
    """
    测试使用有效凭据登录成功

    Given: 数据库中存在一个已激活的用户
    When: 用户提交正确的用户名和密码
    Then: 返回访问令牌和用户信息
    """
    pass
```

### 7.3 调试测试

```bash
# 显示 print 输出
pytest tests/ -v -s

# 进入失败测试的 PDB
pytest tests/ --pdb

# 只运行最后失败的测试
pytest tests/ --lf

# 在第一个失败处停止
pytest tests/ -x

# 显示完整的错误堆栈
pytest tests/ -vv
```

## 九、安全测试（可选）

### 9.1 SQL 注入测试

```python
def test_prevent_sql_injection(client, auth_headers):
    """测试防止 SQL 注入"""
    response = client.get(
        "/api/v1/files/read?path=/etc/passwd",
        headers=auth_headers
    )
    assert response.status_code in [400, 403]
```

### 9.2 XSS 测试

```python
def test_prevent_xss_in_input(client, admin_auth_headers):
    """测试防止 XSS 攻击"""
    response = client.post(
        "/api/v1/users",
        headers=admin_auth_headers,
        json={
            "username": "<script>alert(1)</script>",
            "password": "Password123"
        }
    )
    # 应该拒绝或转义特殊字符
    assert response.status_code == 400
```
